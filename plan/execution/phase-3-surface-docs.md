## 本次不要做

- 不修改 `references/*`、`scripts/*`、`tests/*` 或 `plan/*` 的其他文件。
- 不在本阶段修补 `planctl` 代码或测试，即使发现相关遗留问题，也只记录到后续 phase。
- 不把 README 扩写成细节型百科文档；方法论细节继续通过链接落到 `references/*`。

## 交付检查

- `SKILL.md` 中已明确 long-form fiction / novel / story bible / revision pass 等触发语义。
- `README.md` 与 `README.zh-CN.md` 的安装示例、技能名称和核心场景互相一致，且与 `phase-fiction-skill` 品牌一致。
- `assets/README.md` 中的视觉说明不再引用 phase-contract 的工程执行含义。
- 对本阶段文件完成至少一轮关键词校验，确认工程主语义退场、fiction 主语义入场。

## 执行裁决规则

- 若发现必须修改白名单之外文件才能完成本阶段，停止实施并报告边界冲突，而不是直接扩面。
- 若文案修改后中英文入口对同一能力的表述不一致，视为未完成。
- 若 README 开始承载大段方法论细节而不再保持概括性，视为偏离本阶段目标。
