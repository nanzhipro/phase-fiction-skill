# Phase 0 执行包

本文件不能单独使用。执行 Phase 0 时，必须同时携带完整的 `plan/common.md` 和 `plan/phases/phase-0-premise-promise.md`。

## 必带上下文

- `plan/common.md`
- `plan/phases/phase-0-premise-promise.md`

## 执行目标

- 把故事承诺落成可读、可复用、可验证的 premise 与 canon 文档。
- 让后续角色阶段无需再争论“这到底是什么小说”。

## 本次允许改动

- `story/premise.md`
- `story/canon/foundations.md`
- `story/canon/story-promise.md`

## 本次不要做

- 不写 `story/cast/**`
- 不写 `story/outline/**`
- 不写 `story/draft/**`
- 不顺手改动 `plan/manifest.yaml` 的 phase 顺序

## 交付检查

- 三个目标文件都已创建。
- premise 文档包含一句话梗概、长梗概、主要悬念问题与结局类型约束。
- foundations 文档包含旧案、当前案、列车逻辑和草原环境四个部分。
- story promise 文档列出读者承诺与禁用偷懒方式，各不少于 5 条。

## 执行裁决规则

- 如果把超自然、梦境或全知作弊引入设定，直接判定无效并回退。
- 如果开始设计角色卡、章节矩阵或正文，视为范围漂移，直接回退到 Phase 0 边界。
- 如果 premise 不能明确写出失败代价，视为目标未完成，不得结束本 phase。
