# Phase 1: Build the household suspect web and marriage engine

## 阶段定位

为《回门之前》建立稳定的人物驱动系统，让婚姻关系、宅门关系和旧案责任链同时成为推动悬疑升级的发动机，而不是彼此分离的副线。

## 必带上下文

- plan/common.md
- Phase 0 已完成，story/premise.md 已锁定故事承诺与隐藏真相形状

## 阶段目标

- 为核心角色建立公开身份、真实目标、恐惧、秘密、可交换资源和暴露代价。
- 明确裴见月与陆子衡的新婚关系如何同时承担亲密拉扯、互相试探和彼此怀疑。
- 搭出当前案嫌疑网络，说明每个关键人物如何与阿绛之死、裴景修之死和春和绣坊火案发生连接。

## 明确不做

- 不在这一轮展开时间线、线索账本或章节设计。
- 不提前写具体场景对白或正文片段。
- 不把人物秘密写成只有作者知道、后续无法公平回收的空白盒子。

## 完成判定

- 核心人物都具备可验证的欲望、恐惧、误信念、筹码和暴露后果。
- `story/canon/relationships.md` 已写清婚姻线、血亲线、雇佣线和旧案线的关键张力。
- `story/canon/suspect-web.md` 能解释为什么至少五个关键人物都足以被合理怀疑，而不靠无根据乱猜。
