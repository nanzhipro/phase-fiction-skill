# Phase 0 执行包

本文件不能单独使用。执行 Phase 0 时，必须同时携带完整的 `plan/common.md` 和 `plan/phases/phase-0-premise-promise.md`。

## 必带上下文

- `plan/common.md`
- `plan/phases/phase-0-premise-promise.md`

## 执行目标

- 产出一份可以直接驱动后续人物、真相和正文写作的 premise 文档，而不是停留在主题口号。
- 把婚事、旧案、回门倒计时和夫妻互疑四条主驱动力压成一个可执行承诺。

## 本次允许改动

- `story/premise.md`

## 本次不要做

- 不修改 `story/cast/**`、`story/canon/**`、`story/outline/**` 或 `story/draft/**`。
- 不把“后续再说”留在核心谜面、终局模型或公平性边界上。
- 不为了显得复杂而堆设定；一切只服务类型 promise 和后续推进。

## 交付检查

- premise 文档包含一句话故事承诺、作品定位、核心谜面、隐藏真相形状、终局揭示模型、封闭空间与倒计时说明。
- 文档明确写出失败代价，而不是只说“事情会更糟”。
- 文档没有 placeholder 或待补标记。

## 执行裁决规则

- 如果婚后危机与旧案回潮没有被写成同一条主线，直接判定无效。
- 如果终局仍依赖神秘幕后新人物或超自然解释，直接判定无效。
- 如果 premise 只剩世界观介绍，没有形成明确的叙事问题与倒计时，直接判定无效。
