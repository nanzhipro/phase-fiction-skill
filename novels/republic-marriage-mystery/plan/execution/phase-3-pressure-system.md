# Phase 3 执行包

本文件不能单独使用。执行 Phase 3 时，必须同时携带完整的 `plan/common.md` 和 `plan/phases/phase-3-pressure-system.md`。

## 必带上下文

- `plan/common.md`
- `plan/phases/phase-3-pressure-system.md`
- `story/premise.md`
- `story/canon/case-file.md`
- `story/canon/timeline.md`

## 执行目标

- 把封闭空间、回门倒计时和社会围猎写成后续章节可以直接调用的规则系统。
- 确保每次推进都伴随新的成本，而不是只是时间流逝。

## 本次允许改动

- `story/canon/closed-circle-rules.md`
- `story/canon/countdown-clock.md`
- `story/outline/pressure-map.md`

## 本次不要做

- 不修改 `story/canon/case-file.md`、`story/canon/clue-ledger.md`、`story/canon/timeline.md`。
- 不提前进入 `story/outline/arc-map.md`、`story/outline/reveal-beats.md`、`story/outline/tension-waves.md`。
- 不写正文草稿。

## 交付检查

- closed-circle 规则覆盖物理限制、礼法限制和程序限制。
- countdown-clock 不是简单日历，而是每一格都写清触发点、角色选择和失败后果。
- pressure-map 至少区分外部、关系、证据和身份四类压力，并说明它们在哪一部分叠加。

## 执行裁决规则

- 如果任何压力只体现在对白威胁上、没有实际代价，直接判定无效。
- 如果封闭空间规则中途可以随便失效，直接判定无效。
- 如果倒计时节点与时间线或礼俗冲突，直接判定无效。
